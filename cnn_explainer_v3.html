🔢
A filter = 9 numbers
Each 3×3 filter is just 9 learned weights. The network adjusts them during training so each filter fires on a different feature.
3×3×1 = 9 weights + 1 bias
📐
Dot product = score
Multiply each of the 9 weights by the pixel beneath it, sum them all up. High result = this patch matches what the filter is looking for.
output = Σ(weight × pixel) + bias
🗺
One filter → one map
After sliding to every valid position, each dot product becomes one pixel. 26×26 positions = a 26×26 feature map showing where the filter fired.
26×26 positions → 26×26 feature map

Why does the output shrink?
28×28 → 26×26

A 3×3 filter can't be centred on an edge pixel — there are no neighbours on that side. So the filter only covers positions from (1,1) to (26,26), giving 26 valid positions per row and column.

What the 64 filters actually learn
Each filter = one feature detector

The network never manually programs these patterns. Through training on thousands of labelled images, each filter independently specialises into a different low-level visual detector.

Important: With 64 filters learning independently, the network discovers an efficient vocabulary of low-level features that together give the richest description of the image for the task at hand.
