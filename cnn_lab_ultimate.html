Max Pooling

Pooling reduces spatial size while keeping the strongest signals. A 2×2 window takes the maximum value.

Interactive
Pooling Visualization
How Max Pooling Works

A 2×2 window slides across the feature map. From each window, only the maximum value survives. This halves the spatial dimensions — from 26×26 to 13×13 — reducing computation and making features more position-invariant.

Press the button to step through the pooling process.
📏

Flatten

The 2D feature map is unrolled into a 1D vector, preparing features for the fully connected layers.

2D → 1D Transformation
Why Flatten?

Dense layers expect a 1D input. Flattening rearranges the grid values row by row into a single long vector. For a 13×13 map, this produces a vector of length 169.

🧠

Dense Layers

Fully connected neurons combine all extracted features to make classification decisions.

Visual
Neural Network Forward Pass
How Dense Layers Work

Each neuron receives inputs from ALL neurons in the previous layer, applies learned weights, adds a bias, and passes through an activation function (ReLU). The final dense layer maps to output classes.

📊

Softmax / Output

Raw scores are converted into class probabilities. The highest probability is the model's prediction.

Interactive

The forward pass is now complete. Softmax converts logits into probabilities that sum to 1. The class with the highest probability becomes the model's prediction.

Bias vs Variance

Understanding underfitting, overfitting, and the sweet spot of good generalization.

Training
Train vs Validation Curves
High Bias

The model is too simple to capture the underlying patterns. Both training and validation performance are poor. The model "underfits" — it has not learned enough from the data.

🎯 The Dartboard Analogy

Imagine each prediction is a dart throw. The bullseye is the true answer. Bias = how far the average throw is from center. Variance = how spread out the throws are.

Dropout

A powerful regularization technique that randomly disables neurons during training to reduce overfitting.

Regularization
Training Mode — Some neurons disabled
Dropout Rate: 30%
How Dropout Works

During training, dropout randomly turns off a fraction of neurons. This forces the network to not rely on any single pathway, building redundancy. It's like training many different sub-networks simultaneously.

During inference, all neurons are active, but outputs are scaled to compensate for the dropped connections during training.

🚫
Without Dropout

Over-relies on specific pathways. Prone to overfitting.

✅
With Dropout

Builds robust features. Better generalization.
