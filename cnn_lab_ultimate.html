Layer 1

Input Tensor

We normalize the uploaded image into a 28 x 28 grayscale tensor. Every value is scaled to 0 to 1 so optimization stays numerically stable.

Input shape: [28, 28, 1]
Deep Dive

Think of this as setting everyone to the same scale before the race starts. Pixels originally range from 0 to 255; we divide by 255 so values become 0 to 1. This helps the model learn faster and with fewer unstable jumps.

Here each pixel has one number (brightness), because this dataset is grayscale. So the input is [28, 28, 1]. If it were RGB, it would be [28, 28, 3], and each filter would need three times more values to read.

Layer 2

Convolution + ReLU

A 3x3 kernel slides over every valid location and computes a dot product (sum of weight x pixel). ReLU then applies max(0, x) to keep useful activations and suppress negatives.

Output: 26 x 26 after valid convolution
Deep Dive

A filter is a tiny 3x3 detector sliding across the image. At every position it asks: "How much does this patch look like what I detect?" Early filters usually learn simple patterns like edges or corners.

ReLU keeps useful signals and drops weak/negative ones: max(0, x). This makes the map cleaner and more focused. Without this non-linearity, stacking many layers would be much less powerful.

Layer 3

Max Pooling

Pooling downsamples activations by taking the largest value in each 2x2 window. This compresses spatial resolution while preserving prominent responses.

26 x 26 -> 13 x 13
Deep Dive

Max pooling looks at a 2x2 block and keeps only the strongest value. So even if an edge shifts by 1 pixel, the key signal often survives. That makes the model more robust to small movement.

It also shrinks data a lot (26x26 -> 13x13), so later layers have less noise and less compute, while still keeping the important clues.

Layer 4

Flatten

Flatten reshapes the pooled feature map into a one-dimensional feature vector. No parameters are learned; it is purely a geometric transform.

13 x 13 -> 169 values
Deep Dive

Flatten does not learn anything. It simply unwraps the feature grid into one long list so dense neurons can read it.

So this is a bridge step: convolution finds local patterns, flatten prepares them in vector form for global decision making.

Layer 5

Dense Layer

Dense neurons combine distributed feature evidence into class-relevant latent representations. This is where local patterns become semantic decision signals.

169 -> 32 hidden features
Deep Dive

Dense neurons combine many clues at once. One neuron may respond to "curved sole + lace texture", another to "long vertical structure", and so on. Together they build class evidence.

This stage is powerful but parameter-heavy, which is why earlier conv + pooling layers first compress the image into meaningful features.

INPUT FEATURES -> HIDDEN REPRESENTATION -> LOGITS
Layer 6

Softmax Prediction

Final logits are converted to a normalized probability distribution using Softmax. The largest probability is selected as the predicted class.

Sum of probabilities = 100%
Deep Dive

Softmax converts raw class scores into probabilities. The key rule: all probabilities add to 1 (or 100%).

Highest probability becomes the prediction, but confidence is not perfect truth. A model can still be confidently wrong, so this should be interpreted as model confidence, not certainty.
